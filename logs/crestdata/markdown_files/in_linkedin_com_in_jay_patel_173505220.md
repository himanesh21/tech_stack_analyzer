Provisioning = Creating user accounts and granting access. When a new employee joins: ✔ Account created in Active Directory / Entra ID ✔ Email configured ✔ Access to applications (CRM, ERP, HRMS, etc.) ✔ Role-based permissions assigned If provisioning is delayed, productivity suffers. But if deprovisioning is delayed? Security suffers. 🚨 What is User Deprovisioning? Deprovisioning = Removing access when someone leaves the organization. This includes: ❌ Disabling Active Directory / Entra ID accounts ❌ Revoking SaaS access (e.g., Salesforce, ServiceNow) ❌ Removing VPN & privileged roles ❌ Rotating shared passwords ❌ Recovering assets This is where most companies fail. 🔎 Real-Life Scenario (Very Common) An employee resigns. HR updates the exit date in the HRMS. But IT is informed 48 hours later. Meanwhile: ⚠ The user still has VPN access ⚠ Still has access to shared drives ⚠ Still has access to client data Now imagine: That employee joins a competitor. This is not a hacker attack. This is an Identity Lifecycle failure. This is exactly what security frameworks like ISO/IEC 27001 and NIST emphasize — timely access revocation. 🎯 Hard Truth in IAM Provisioning impacts productivity. Deprovisioning impacts security, compliance, and reputation. One delayed provisioning ticket = 2 hours of lost work. One delayed deprovisioning = potential data breach. Which one sounds riskier?
